# Handoff — V2 Visual Edition (Diamond/Pearl-grade pass)

This round replaced the placeholder `ColorRect` art with a full pixel-art layer
and rebuilt Konoha as a real scrolling village. Everything is procedurally
generated by a Python toolchain so it can be re-rolled and extended.

## What changed

| Area | Before | Now |
|---|---|---|
| Resolution | 240×160 | **256×192** (DS-native), window 1024×768, integer scaled |
| Overworld | flat color squares | real tileset, **Camera2D** follow, animated 4-dir walk sprites, building/prop objects, depth sort |
| Konoha | one 10×15 screen | **44×32 scrolling village** matching the FireRed reference: Hokage Rock (4 carved faces), Hokage Tower, Academy, Hospital, Tool Shop, **2 Ichiraku Ramen heal-stalls**, houses, plaza, river+bridge, south gate, Forest-of-Death exit |
| Pokémon Center | n/a | **Ichiraku Ramen stalls are the team-heal points** (red noren, steaming bowls, lanterns) |
| Battle | 32px squares on flat bg | sky/treeline backdrop + platforms, **D/P framing** (player back-view, enemy front-view), DS-style HUD with gradient HP/CP bars, message + action menu |
| Battle input | **frozen** (no input handler) | fixed — `BattleController._unhandled_input` routes to the menu |
| Title | plain text on navy | Hokage-Rock establishing-shot backdrop + outlined logo + DS menu |
| Roster art | none | **39 character/NPC walk-sheets** (31 units + NPCs), canon-accurate |
| Other maps | 10×15 | Training Grounds (sand arena, three logs, **Memorial Stone**) and Forest of Death (giant-tree maze, **central tower**, chain-link Gate 44, stream) rebuilt |

## Asset pipeline (all in `tools/art/`, pure Python + Pillow)

- `pixutil.py` — canvas/primitives, palette, selective silhouette outline.
- `ninja.py` — parametric chibi walk-cycle generator (4 dir × 3 frames, 24×32).
- `roster.py` — per-character spec table (canon colours + signature features).
- `tiles.py` — 16×16 terrain tile atlas (grass/path/water+shores/forest/cliff/…).
- `buildings.py` — building + prop object sprites (ramen, Hokage Tower, gate, …).
- `battle_assets.py` — creature battle sprites, backdrops, platforms, jutsu FX.
- `map_konoha.py` / `map_others.py` — procedural map builders → `data/maps/*.json`.
- `compositor.py` — renders a full map screen from the real assets (preview/QA).
- `export_overworld.py`, `export_world.py` — write PNGs + `data/visuals/*.json`.

Regenerate everything:

```bash
cd tools/art
python export_overworld.py && python export_world.py && python battle_assets.py
python map_konoha.py && python map_others.py
# then re-import in Godot:  godot --headless --path . --import
```

## Engine wiring

- `data/visuals/{units,tiles,buildings}.json` resolved by `DataRegistry`
  (`unit_visual`, `building_visual`, `tiles_visual`). **All paths fall back to
  the old ColorRect rendering if a texture is missing** — the game never breaks
  mid-rollout.
- New map schema adds `size`, `spawn`, `buildings[]`, `props[]`; ground `grid`
  uses the legend in `tiles.json`. Building footprints auto-block; NPCs at doors
  drive heal/shop/boss actions (unchanged handlers).
- `src/ui/UiKit.gd` — shared DS palette + panel/label/bar builders.

## Notes / gotchas

- Godot's "safe save" can intermittently fail to write a `.ctex` on this machine
  (antivirus/file lock). Re-running `--import` 2–3× resolves it; if one file
  stays stuck, rename it (fresh import path) — that's why the wild-battle backdrop
  is `battle_bg_wild.png`.
- Side-view walk frames are semi-frontal with a profile nose; fine at play scale.
- Detailed front-pose sprites sliced from `konoha11_roster_sheet.png` live in
  `tools/art/_sliced/` — unused in-engine (kept for future menu portraits).
